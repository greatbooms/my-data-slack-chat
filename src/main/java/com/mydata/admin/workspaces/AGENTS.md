# 에이전트 안내

이 패키지는 관리자 GraphQL에서 사용하는 워크스페이스 관리 로직과 payload/input 모델을 담당합니다.

## 핵심 책임

- 워크스페이스 생성, 이름 변경, 소유자 변경
- 워크스페이스 소프트 삭제와 복구
- 사용자 생성/부트스트랩 시 기본 `Personal` 워크스페이스 보장
- 관리자 UI와 GraphQL schema가 사용할 payload/input 형태 제공

## 주의사항

- 워크스페이스 소프트 삭제는 단순 목록 숨김이 아니라 하위 데이터소스 수집과 검색 범위에도 영향을 줘야 합니다.
- 삭제된 사용자는 새 워크스페이스 소유자가 될 수 없습니다.
- 삭제된 워크스페이스는 일반 목록과 데이터소스 생성 옵션에서 제외합니다.
- 복구는 기존 식별자를 유지하되, 필요한 경우 하위 데이터소스/검색 범위 영향도 함께 검토합니다.
- 새 mutation을 추가하면 `src/main/resources/graphql/admin.graphqls`, frontend GraphQL 문서, generated 타입, 통합 테스트를 함께 갱신합니다.
